# macXterm 使用者指南

歡迎使用 **macXterm** —— 一款快速、原生、跨平台的終端機與遠端運算工具箱。如果你曾在
Windows 上用過 MobaXterm，並希望在 macOS 或 Linux（以及 Windows）上也能擁有同樣的分頁式
終端機、SSH/SFTP、通道與網路工具，macXterm 的設計就是要讓你感到熟悉，同時保持開放、
MIT 授權，且沒有人為限制。

本指南面向**終端使用者**，將帶你了解應用程式視窗中可以做的一切。若要從原始碼建置，請參閱
[`BUILD.zh.md`](BUILD.zh.md)。

---

## 1. macXterm 是什麼、適合誰使用

macXterm 是一款單一的桌面應用程式，把開發者、系統管理員或網路工程師每天都會用到的工具
整合在一起：

- 具備真正 VT100/VT220/xterm 模擬的**分頁式終端機**。
- 支援多種協定的**遠端連線**：SSH、Telnet、Serial、Mosh、RSH、Rlogin、
  XDMCP、RDP、VNC —— 再加上一個純本機的 **Shell**。
- 建構於你的 SSH 連線之上的圖形化 **SFTP 檔案瀏覽器**。
- **SSH 通道**（local、remote、dynamic/SOCKS）與跳板主機（jump host）支援。
- **MultiExec** —— 只需輸入一次，即可同時廣播到多個面板。
- 由主密碼保護的加密**憑證保險庫**。
- 一組內建的**網路工具**：連接埠掃描器、子網掃描、封包擷取、
  金鑰產生、圖片檢視器、文字/資料夾差異比對、遠端系統監控器，以及輕量的
  TFTP / HTTP / FTP / Telnet / CRON / NFS / SSH 伺服器。
- 能整合你所在平台 X 伺服器的 **X11 轉發**功能。

它以原生 **Qt + C/C++** 撰寫，並以同一份原始碼在 **Windows、macOS 與 Linux** 上執行，
並以 **MIT 授權**發布。不同於 MobaXterm 的免費「Home」版，macXterm 對已儲存的工作階段、
通道或巨集數量**沒有任何上限**。

## 2. 安裝與啟動

### 安裝

macXterm 會建置成單一執行檔。若你的平台已有預先建置好的執行檔，直接執行即可。若要自行
建置，請依照 [`BUILD.zh.md`](BUILD.zh.md) 的指示 —— 簡而言之，你需要 Qt 6、
OpenSSL 3.2+、libvterm 與 libssh2，然後進行標準的 CMake 設定與建置流程。

### 首次啟動

macXterm 啟動後，你會看到一個分為三個部分的視窗：

- **工作階段樹側邊欄**（左側）—— 可停駐的「Sessions」面板，列出你儲存在根資料夾下的
  連線設定檔。雙擊（啟動）任一項目即可開啟它。頂端的**篩選方塊**會隨著輸入即時縮小樹狀
  結構，比對工作階段名稱、主機、使用者名稱或資料夾（不區分大小寫）；沒有符合項目的資料夾
  會被排除，清空方塊即可還原完整樹狀結構。在任一列上按**右鍵**可叫出編輯功能選單：於
  **工作階段**上——開啟、開啟 SFTP 瀏覽器（SSH/SFTP）、編輯、重新命名、複製、設定圖示、
  移動到資料夾（含就地的「新資料夾…」）、複製主機位址／複製 SSH 指令，以及刪除；於
  **資料夾**上——在此新增工作階段、重新命名資料夾、移除資料夾（保留其中的工作階段），
  以及全部展開／收合；於**根節點**上——新增工作階段與全部展開／收合。
- **分頁式終端機區域**（中央）—— 每個開啟的工作階段都各自位於一個可關閉、可移動的分頁中。
- **工具列**（頂端）—— 主要動作列，包含以下按鈕：
  - **New Shell** —— 開啟另一個本機 shell 分頁。
  - **New Session** —— 透過對話框建立/開啟遠端工作階段。
  - **MultiExec** —— 切換是否將按鍵廣播到多個面板。
  - **Tunnel** —— 開啟 SSH 通道對話框。
  - **Settings** —— 開啟偏好設定對話框。
  - **Vault** —— 解鎖（或設定）你的加密憑證保險庫。

啟動時，macXterm 會自動開啟**一個本機 shell 分頁**，讓你立即擁有可用的提示字元 ——
無需任何設定。

## 3. 工作階段類型與建立方式

macXterm 透過**工作階段（session）**連線到遠端系統。點選工具列上的 **New Session** 即可
開啟工作階段對話框，其中包含以下欄位：

| 欄位 | 意義 |
|-------|---------|
| **Name** | 工作階段的標籤；同時也是分頁標題與側邊欄項目名稱。 |
| **Type** | 使用的協定（詳見下方）。 |
| **Host** | 遠端主機的主機名稱或 IP 位址。 |
| **Port** | TCP/序列埠。留空（0）則使用該協定的預設值。 |
| **Username** | 登入名稱（適用於需要驗證的協定）。 |

點選 **OK** 後，該工作階段會經過驗證、儲存到側邊欄樹狀結構，並在新分頁中開啟。

> 若 Port 欄位留空，**預設連接埠**會自動填入：SSH/Mosh 為 **22**、
> Telnet 為 **23**、Rlogin 為 **513**、RSH 為 **514**、RDP 為 **3389**、
> VNC 為 **5900**、XDMCP 為 **177**。

### 支援的類型

New Session 對話框提供 **14 種類型**：SSH、Telnet、Serial、Mosh、RSH、Rlogin、
XDMCP、SFTP、FTP、S3、RDP、VNC、Browser，以及本機 Shell。對話框分為
**General / Advanced / Terminal** 三個分頁，只會顯示與所選類型相關的欄位。

- **SSH** —— 主力協定。需要 **Host**、**Username**，並可選填 **Port**。
  驗證方式可以是密碼（於連線時提供，通常來自憑證保險庫）或私鑰（`keyfile` 參數，
  可選填密碼片語）。同一個 SSH 工作階段也承載 SFTP、通道與 X11 轉發。
- **Telnet** —— 具選項協商功能的純 Telnet。需要 **Host**（若不是 23 則也需 **Port**）。
- **Serial** —— 用於硬體的序列主控台。以序列裝置作為連接埠，線路設定（鮑率、資料位元、
  同位元、停止位元、流量控制）預設為 **9600 8N1**。
- **Mosh** —— 行動化 shell，在 SSH 引導之上具備漫遊友善特性。需要 **Host**/**Username**。
- **RSH** / **Rlogin** —— 建立在簡單 TCP 串流上的舊式遠端 shell 協定，各自有其啟動
  交握流程。需要 **Host**（登入交握還需要 **Username**）。
- **XDMCP** —— X Display Manager 查詢引導。需要 **Host**。
- **SFTP** —— 專屬的 SFTP 檔案瀏覽器工作階段（無終端機）。需要 **Host**/**Username**。
- **FTP** —— 圖形化 FTP 瀏覽器（被動模式）。需要 **Host**。
- **S3** —— Amazon S3 儲存桶瀏覽器（SigV4 簽章）。需要儲存桶與憑證。
- **RDP** —— 連往 Windows 主機的遠端桌面。需要 **Host**（以及 **Username**）；
  預設連接埠為 3389。**Advanced** 分頁可設定解析度、剪貼簿/磁碟機/音訊重新導向與 NLA。
- **VNC** —— 遠端畫面緩衝區／螢幕分享。需要 **Host**；預設連接埠為 5900。完全互動
  （滑鼠 + 鍵盤），Advanced 分頁提供**唯讀（view-only）**切換開關。
- **Browser** —— 內嵌的網頁檢視器，附有網址列。
- **Shell** —— 在你自己機器上的**本機**命令列 shell。不需要主機/使用者名稱；這就是
  啟動分頁所使用、也是 **New Shell** 所開啟的類型。

SSH 的 **Advanced** 分頁另外提供壓縮、X11 轉發、代理驗證/轉發、**keepalive** 間隔、
用來取代登入 shell 的**遠端命令**，以及「命令結束後仍保持面板開啟」選項。每個工作階段
也都能歸類到某個**資料夾**並指定顯示用的**圖示**（General 分頁），終端機類型的工作階段
還可以在每個書籤上覆寫全域的字型/配色方案/回捲/Backspace 設定（**Terminal** 分頁）。

## 4. 使用終端機

每個工作階段都會在自己的**分頁**中開啟。分頁**可關閉**（分頁上的 ×，或按
`Ctrl+Shift+W`），也**可移動**（拖曳以重新排序）。使用 `Ctrl+Tab` /
`Ctrl+Shift+Tab` 切換分頁。

### 模擬

終端機由真正的 **VT100/VT220/xterm** 模擬器驅動（建構於 libvterm 之上），因此全螢幕的
curses 應用程式 —— `vim`、`htop`、`tmux`、`less` 及其他同類程式 —— 都能正確呈現，
包括 256 色**及真彩色**、游標移動與裝置狀態回應。表情符號與其他星空平面（astral-plane）
字符能完整顯示，且無論在本機或遠端工作階段，你都可以**透過輸入法輸入中日韓文
（CJK）**。

### 更多終端機功能

- **分割面板** —— 從 View 選單將分頁分割為 2 直、2 橫或 2×2。
- **右鍵選單** —— 複製 / 貼上 / 全選 / 清除回捲紀錄 / 尋找。
- **回捲紀錄搜尋** —— `Cmd`/`Ctrl+Shift+F` 開啟涵蓋歷史紀錄的搜尋列。
- **開啟網址** —— 對輸出中的連結按住 `Cmd`/`Ctrl` 點擊，即可在瀏覽器中開啟。
- **滑鼠回報** —— 要求滑鼠事件的應用程式（tmux、vim、htop）能夠接收到相關事件。
- **括號式貼上（Bracketed paste）**及可選的**貼上延遲**，適用於較慢的遠端主機；
  多行貼上會在傳送前發出警告。
- **工作階段記錄** —— 將工作階段的輸出鏡射到檔案。

### 配色方案

內建三種配色方案，可於 **Settings → Terminal → Color scheme** 中選擇：

- **Dark**（預設）—— 黑底淺灰字，標準 xterm 16 色 ANSI 調色盤。
- **Light** —— 白底黑字。
- **Solarized Dark** —— 熟悉的 Solarized 背景/前景配色，並經過調整的基礎色彩。

### 字型

在 **Settings → Terminal** 中設定**字型**家族與**字型大小**（6–72 pt）。你也可以在此
設定**回捲（Scrollback）**緩衝區（保留的行數，最多 1,000,000 行）。Powerline / Nerd Font
的提示字符會自動回退到已安裝的 Nerd Font，因此使用主題化 shell 提示時不會顯示為方框
（tofu）。以上任一設定都可以在 New Session 對話框的 **Terminal** 分頁中**針對個別
工作階段覆寫**。

## 5. MultiExec —— 將輸入廣播到多個面板

**MultiExec** 讓你能同時操控多個工作階段：在一處輸入命令，該命令便會傳送到每個參與的
終端機 —— 非常適合在一整批伺服器上執行相同操作。

- 點選工具列上的 **MultiExec** 按鈕以切換廣播模式的開啟/關閉。
- 啟用期間，按鍵輸入會傳送到**每一個已啟用的目標**面板。
- 個別面板可以**選擇退出**廣播，已關閉的面板也會自動被略過，因此你不會意外把內容輸入到
  已結束的工作階段中。

MultiExec 對於參與的面板數量**沒有上限**。

## 6. SFTP 與 FTP 檔案瀏覽器

每個 SSH 工作階段都會在同一條已驗證的連線上開啟一個圖形化 **SFTP** 側邊面板 —— 無需
第二次登入 —— FTP 工作階段也會在 FTP 連線上開啟同樣的面板。此瀏覽器停駐在左側
（與 MobaXterm 一致），並讓你可以：

- **瀏覽**遠端目錄，含 Name / Size / **Modified** 欄位與可排序的表頭；可跳轉到
  **Home** 或上一層目錄。
- **下載／上傳**：可透過按鈕，也可**拖放**——將作業系統的檔案拖放到面板上即可上傳，
  或將遠端檔案拖出面板即可下載。整個資料夾可**遞迴**傳輸，並附有可取消的進度條。
- **就地編輯遠端檔案** —— 雙擊檔案即可在內建編輯器中開啟；儲存時會**自動重新上傳**。
- **管理** —— 右鍵可執行 chmod / 重新命名 / 刪除 / 新增資料夾。
- **跟隨終端機所在目錄** —— 啟用後，瀏覽器會（透過 OSC 7）自動切換到終端機當前所在的
  目錄。

該面板會在**其所屬工作階段的分頁關閉時自動關閉**。實際瀏覽需要真實的遠端端點；路徑
處理、清單列出與遞迴傳輸邏輯則有獨立的單元測試涵蓋（SFTP 針對真實的 `sshd` 測試；
FTP 則針對內嵌伺服器測試）。

## 7. SSH 通道與跳板主機

點選工具列上的 **Tunnel** 以開啟 SSH 通道對話框。選擇一種 **Type** 並填入端點資訊：

| 類型 | 作用 | 使用的欄位 |
|------|--------------|-------------|
| **local** | 在本機的 `bind:bindPort` 上監聽，並透過 SSH 轉發到 `target:targetPort`。 | Bind address, Bind port, Target host, Target port |
| **remote** | 在 SSH **伺服器**上監聽，並將連線轉發回你這端可觸及的 `target:targetPort`。 | Bind address, Bind port, Target host, Target port |
| **dynamic** | 在 `bind:bindPort` 上執行一個 **SOCKS 代理**（無固定目標）。 | Bind address, Bind port |

對話框預設的 bind 位址為 `127.0.0.1`。當通道有 bind port，且（對 local/remote 而言）
有目標主機與連接埠時，即視為有效。通道管理器會拒絕無效的通道，並拒絕 **bind port
衝突**，因此兩條通道不會爭用同一個本機連接埠。你可以定義的通道數量**沒有限制**。

### 跳板主機（Jump hosts）

macXterm 支援**跳板主機**（SSH `ProxyJump`）。當你匯入指定了 `ProxyJump` 的 OpenSSH
設定檔時，跳板主機資訊會被保留在該工作階段中（作為其 `jumphost` 參數），因此連線會
透過堡壘主機（bastion）路由。

## 8. 憑證保險庫與主密碼

macXterm 將你的機密資訊（例如 SSH 密碼）儲存於加密的**憑證保險庫**中，並由單一的
**主密碼**解鎖。點選工具列上的 **Vault**：

- **Set Master Password**（首次使用／建立模式）—— 輸入至少 **8 個字元**的主密碼並確認。
- **Unlock Vault** —— 輸入主密碼以解密該工作階段所儲存的機密資訊。

### 你的機密資訊如何受到保護

此保險庫使用的是強健、現代的加密技術 —— 而非單純的資料模糊化：

- 主密碼會以 **Argon2id**（記憶體密集型 KDF，64 MiB / 3 次迭代）延展成 256 位元金鑰。
  在不支援 Argon2id 的較舊系統上，會透明地回退到 **scrypt**；保險庫會記錄所使用的 KDF，
  因此在任何建置版本上都能正確解密。
- 機密資訊以 **AES-256-GCM** 加密，這是一種具驗證機制的加密演算法 —— 若檔案遭竄改
  或密碼錯誤，解密會乾淨地失敗，而不會回傳亂碼。
- 每次儲存都會產生隨機的**鹽值（salt）**與**隨機數（nonce）**，衍生出的金鑰與解密後的
  明文在使用完畢後會從記憶體中清除。

當 SSH 工作階段建立連線時，保險庫可以注入已儲存的密碼，讓你不需要重新輸入。

## 9. 設定

從工具列開啟 **Settings**，其中包含兩個分頁：

### Terminal
- **Font** —— 終端機使用的字型家族。
- **Font size** —— 6 至 72 pt。
- **Color scheme** —— Dark、Light 或 Solarized Dark。
- **Scrollback** —— 保留的行數（0 – 1,000,000）。

### X11
- **Auto-start X server** —— 在需要時自動啟動平台的 X 伺服器。
- **Enable SSH X11 forwarding** —— 透過 SSH 將遠端 GUI 應用程式的畫面通道回你的顯示器。

## 10. 鍵盤快捷鍵

以下是預設綁定。在 macOS 上，`Ctrl` 會自動對應到 `⌘`，因此同一份定義可以跨平台使用。

| 動作 | 快捷鍵 |
|--------|----------|
| 新增終端機 | `Ctrl+Alt+T` |
| 關閉終端機 | `Ctrl+Shift+W` |
| 下一個分頁 | `Ctrl+Tab` |
| 上一個分頁 | `Ctrl+Shift+Tab` |
| 分離分頁 | `Ctrl+Shift+D` |
| 切換全螢幕 | `F11` |
| 貼上 | `Shift+Ins` |
| 快速連線 | `Ctrl+Shift+Q` |
| 切換 X11 | `Ctrl+Shift+X` |
| 開啟編輯器 | `Ctrl+Shift+M` |

快捷鍵可以**重新綁定**；若新綁定會與已指派的動作衝突，登記系統會拒絕該綁定。

## 11. 匯入既有工作階段

你不需要重新建立已經存在的連線設定。

### OpenSSH（`~/.ssh/config`）

macXterm 可以直接匯入你的 OpenSSH 用戶端設定檔。它會讀取每個 `Host` 區塊並依此建立
SSH 工作階段，對應方式如下：

- `Host` → 工作階段名稱
- `HostName` → 主機（若省略則回退為使用該別名）
- `User` → 使用者名稱
- `Port` → 連接埠
- `IdentityFile` → 私鑰檔案（`keyfile`）
- `ProxyJump` → 跳板主機（`jumphost`）

萬用字元樣式（`Host *`、`?`）會被略過，`=` 與空白分隔符號皆能正確處理。匯入的工作階段
會歸類到 **"Imported (ssh_config)"** 資料夾下。

### PuTTY

macXterm 的工作階段儲存格式採用與 PuTTY 已儲存工作階段配置一致的 INI 風格格式。完整的
PuTTY 匯入功能正在規劃中，將會沿用與 OpenSSH 設定檔相同的匯入路徑。

## 12. 內建工具

macXterm 內建了一些小型網路工具，讓你不需要為此另外開啟其他應用程式。這些工具都不受
MobaXterm Home 版所施加的執行期上限限制。

- **連接埠掃描器** —— 基於 TCP-connect 的掃描器。可檢查單一主機/連接埠，或掃描一段
  含首尾的**連接埠範圍**，找到的每個開放連接埠會即時回報，並在最後給出開放連接埠總數。
- **子網掃描** —— 對一段 CIDR 範圍進行 ping/探測，以找出存活的主機。
- **封包擷取** —— 基於 `libpcap` 的擷取功能，附帶內建封包解碼器（需要對該網路介面
  具有擷取權限）。
- **金鑰產生** —— 產生並顯示 SSH 金鑰指紋（MobaKeyGen 風格）。
- **圖片檢視器** —— 具備上一張/下一張導覽功能的全螢幕圖片檢視器。
- **文字與資料夾差異比對** —— 基於行層級、LCS 演算法的文字差異比對，加上遞迴式資料夾
  比對（MobaTextDiff / MobaFoldersDiff 風格）。
- **配色方案編輯器** —— 圖形化的 ANSI 調色盤／前景背景色編輯器。
- **輕量伺服器**（工具列 → Servers），每一項都不受 MobaXterm Home 版執行期上限限制：
  **TFTP**（RFC 1350）、**HTTP GET** 檔案伺服器、**FTP**（RFC 959，具完整被動資料通道）、
  **Telnet**、**CRON**（5 欄位排程表達式）、**NFSv3（可讀寫）**，以及一個
  **SSH/SFTP** 伺服器（密碼驗證 + PTY shell）。
- **遠端監控器** —— SSH 終端機下方的狀態列，顯示從遠端主機解析出的即時 CPU / RAM /
  NET 資訊。

## 13. X11 轉發

macXterm 可以透過 SSH 將**遠端圖形化（X11）應用程式**顯示在你的桌面上。它**不會**
內建自己的 X 伺服器，而是整合**所在平台的 X 伺服器**，並管理提供給被轉發應用程式的
`DISPLAY`：

- **macOS** —— **XQuartz**
- **Windows** —— 如 **VcXsrv** 這類 X 伺服器
- **Linux** —— 你的**原生 X.Org** 伺服器

在 **Settings → X11** 中啟用 **SSH X11 forwarding**，並可選擇搭配 **Auto-start X
server**，接著在 SSH 工作階段中執行 GUI 程式 —— 它會以本機視窗的形式出現。
`Ctrl+Shift+X` 可快速切換 X11。

## 14. 目前狀態與限制

macXterm 對於哪些功能需要真實的對端、哪些功能完全能在你的機器上運作，抱持坦誠的態度。

**完全可在本機運作（不需要伺服器）：**
- 本機 **Shell** 分頁、VT100/xterm 模擬（真彩色、表情符號、CJK 輸入）、分割面板
- 配色方案、字型（附 Nerd Font 回退機制）、回捲紀錄、每個工作階段的覆寫設定、Settings
- **憑證保險庫**（AES-256-GCM + Argon2id/scrypt）以及資料夾/圖示式工作階段樹
- **MultiExec** 廣播；**巨集**與**快捷鍵**
- **文字/資料夾差異比對**、**圖片檢視器**、**連接埠掃描器**、**子網掃描**，以及輕量的
  **TFTP / HTTP / FTP / Telnet / CRON / NFS / SSH** 伺服器
- OpenSSH `~/.ssh/config` 與 `MobaXterm.ini` **匯入**
- 通道與工作階段的**驗證**（連接埠衝突與欄位檢查）

**需要真實的遠端（或硬體）才能完整運作：**
- **SSH**，以及建構在其上的 **SFTP** 瀏覽器、**通道**、**X11 轉發** ——
  這些都需要真實的 `sshd`
- **Telnet、Mosh、RSH、Rlogin** —— 需要可連線的伺服器
- **FTP / S3** —— 需要可連線的 FTP 伺服器／S3 儲存桶
- **Serial** —— 需要真實的序列裝置
- **RDP / VNC** —— 需要遠端桌面／VNC 主機
- **遠端監控器** —— 需要可供取樣的 SSH 主機
- **封包擷取** —— 需要在真實網路介面上具有擷取權限

**已知的缺口。** **XDMCP** 已完成探索/協商交握，但尚未將接受後的工作階段重新導向到
本機 X 伺服器（需要真正的顯示管理器）；**VNC** 可解碼 Raw/CopyRect/RRE/Hextile/ZRLE
（尚未實作 Tight）；在 **Windows** 上，ConPTY 本機 shell 尚未完成。僅限 Windows 的
MobaXterm 功能（WSL、Cygwin shell 擴充、MobApt、PuTTY 登錄檔匯入）依設計刻意排除於
macOS/Linux 之外。

**沒有人為限制。** 不同於 MobaXterm Home 版，macXterm 對已儲存工作階段、通道或巨集
數量**沒有上限**，對 MultiExec 面板數量或內建伺服器也沒有上限。它採 MIT 授權，可自由
使用而不受限制。
